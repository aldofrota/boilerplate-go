# Boilerplate Go

O **Boilerplate Go** é uma estrutura base para desenvolvimento de aplicações em Go, configurada para acelerar o início de novos projetos ao fornecer uma arquitetura pré-definida e melhores práticas integradas. Esta base inclui configuração de containers Docker, gerenciamento de dependências, integração com Redis e MongoDB, além de ferramentas essenciais para facilitar o desenvolvimento, build e testes.

## Funcionalidades Principais

- **Configuração de Containers**: Docker e Docker Compose configurados para fácil integração com Redis e MongoDB, prontos para uso em ambiente local e produção.
- **Estrutura de Código Modular**: Organizado em camadas para suportar uma separação clara entre responsabilidades como controllers, services e repositórios.
- **Suporte para API RESTful**: Rotas básicas configuradas e arquitetura REST para rápida implementação de endpoints.
- **Documentação com Swagger**: Inclusão de Swagger para documentação automática da API, simplificando a integração e o uso dos endpoints.
- **Utilitários para Desenvolvimento**: Scripts e ferramentas para facilitar o ciclo de vida do desenvolvimento, incluindo Air para live reloading e configuração do Go para builds otimizados.
- **Gerenciamento de Erros e Logs**: Tratamento centralizado de erros e logs estruturados para facilitar o monitoramento e a depuração.
- **Base para Testes Automatizados**: Configuração inicial para testes com `gotestsum`, incentivando a cobertura de código desde o início.

## Objetivo

O objetivo do **Boilerplate Go** é fornecer uma base sólida que poupe tempo de configuração e permita que desenvolvedores Go se concentrem nas funcionalidades específicas do projeto, reduzindo a necessidade de configurar aspectos comuns de infraestrutura e arquitetura.

---

Esse boilerplate é ideal para qualquer projeto de API ou serviço em Go que precise de uma estrutura base consistente, facilitando a escalabilidade e a manutenção ao longo do tempo.
